Containers In Bootstrap

Normal container class (.container)

This is the most basic container in Bootstrap. It sets the max-width at each responsive breakpoint. In other words, it has a fixed width that changes at different breakpoints.

Fluid Container (.container-fluid)

This container will always be 100% width no matter what - it sets the width: 100% at all breakpoints. It will always span the entire length of the viewport. Here's some sample text to demonstrate:

Lorem, ipsum dolor sit amet consectetur adipisicing elit. Illum officiis eos eum explicabo assumenda. Quam laudantium tempora necessitatibus. Corrupti nisi illum tempore aliquam fugit. Reiciendis rem sit aliquid quae harum?

Breakpoint Containers (.container-{breakpoint})

Also known as "Responsive Containers", this class sets a responsive, fixed-width container that activates at the specified breakpoint and wider. Width: 100% until the specified breakpoint. Before that breakpoint, it behaves like fluid containers (.container-fluid). It will span the entire length of the viewport. However, once it hits the specified breakpoint, the container jumps back to behaving as a normal container class, where it gets assigned a max-width and will not extend past this width.

This particular example paragraph demonstrates a container, set with a large screen breakpoint, with a value of 5 for vertical margins (the class is .container-lg). This means that it will have 100% width until lg screens, then it will assume the behavior of .container, assigning a max-width.

Because Bootstrap's grid system is built on Flexbox, you can leverage Flexbox utility classes directly on your grid elements. You can use flex utilities like justify-content to control the horizontal spacing of flex items inside a flex container.

In the example shown above, the justify-content utility class is used on the flex container (.row) : div class="row justify-content-end" This creates 3 immediate flex children - the 3 columns. The -end after justify-content tells the browser to shift the flex items to the end of the container.

Aligning Items

col 1
col 2
col 3

The align-items utility is used to control the vertical alignment of flex items inside a flex container. Remember, you must have .d-flex or .row classes to make it a flex container and allow align-items to work properly.

Code from above example: div class="row align-items-end";

Here the div that's the flex row has align-items-end declared on it. This makes it so that all flex children (col 1, col 2, and col 3) are aligned vertically at the end of the flex container.

Modals

Next, let's talk about Modals. Modals are a Bootstrap component that manifests as a pop-up dialog box that appears over the main webpage content to grab the user's attention for a specific task or message.

Modals overlay the page with a dimmed background to focus on the modal content. They can contain text, images, forms, or interactive content. Modals are dismissed by either clicking the close button, outside the modal, or pressing the Esc key.

When the button "Register for Updates" below is clicked, it will trigger the modal to pop-up. This is because we have the data-bs-toggle and dad-bs-target attributes assigned to that button.

There are no nested modals - Bootstrap only supports one modal at a time.
