Normal container class (.container)

This is the most basic container in Bootstrap. It sets the max-width at each responsive breakpoint. In other words, it has a fixed width that changes at different breakpoints.

Fluid Container (.container-fluid)

This container will always be 100% width no matter what - it sets the width: 100% at all breakpoints. It will always span the entire length of the viewport. Here's some sample text to demonstrate:

Lorem, ipsum dolor sit amet consectetur adipisicing elit. Illum officiis eos eum explicabo assumenda. Quam laudantium tempora necessitatibus. Corrupti nisi illum tempore aliquam fugit. Reiciendis rem sit aliquid quae harum?

Breakpoint Containers (.container-{breakpoint})

Also known as "Responsive Containers", this class sets a responsive, fixed-width container that activates at the specified breakpoint and wider. Width: 100% until the specified breakpoint. Before that breakpoint, it behaves like fluid containers (.container-fluid). It will span the entire length of the viewport. However, once it hits the specified breakpoint, the container jumps back to behaving as a normal container class, where it gets assigned a max-width and will not extend past this width.

This particular example paragraph demonstrates a container, set with a large screen breakpoint, with a value of 5 for vertical margins (the class is .container-lg). This means that it will have 100% width until lg screens, then it will assume the behavior of .container, assigning a max-width.
